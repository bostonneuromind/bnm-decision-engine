베타 운영 가이드

Boston Neuromind LLC · 파트타임 세라피 클라이언트 5명

베타 운영 원칙
1. 시스템 추천은 제안일 뿐, 임상가가 최종 결정
2. 안전 catch (Crisis flag, mandated reporter) 항상 우선
3. 모든 outcome 데이터 기록 → 시스템 학습
4. 데이터 prediction error 누적되면 매핑 행렬 조정 고려

1. 베타 클라이언트 자격 기준

포함 기준

  • 18세 이상
  • 학습 효율 / 수행 최적화 / 가벼운 불안·우울 영역
  • 최소 8주 참여 가능 (4-8주 프로토콜 + 4주 outcome)
  • 한국어 또는 영어 의사소통 가능
  • 인지·바이오마커 측정 가능 (Polar H10 또는 Apple Watch, CPT 등)

제외 기준

  • 급성 정신증
  • 활성 자살 위기 (Columbia SSRS 필수)
  • 통제되지 않은 양극성 장애
  • 최근 6개월 자살 시도력
  • 약물 의존 (현재)

2. 회기 1 워크플로 (75-105분)

  1. Step 1: 클라이언트 등록 (5분) — 이니셜, ID, 주요 호소
  2. Step 2: Entry Protocol 5 (30분) — 시간추정, HRV, CPT, N-back, VSR
  3. Step 3: 증상 데이터 수집 (15분) — PHQ-9, GAD-7, 주관적 점수
  4. Step 4: 시간 정보 인터뷰 (15분) — v7 NEW — 8문항
  5. Step 5: 클라이언트 목표 (10분) — 영역 선택, 동기, 장애물
  6. 시스템 실행 (5분) — /clinician.html
  7. 결과 검토 + 클라이언트 설명 (15분) — DDE, Decomposition 함께 설명

3. 4주 회기 운영

영역별 평가:

영역기간회/주진행
주의력4주3회Complexity ramp
학습 효율4주4회Consolidation
수행 최적화4주3회Skill stacking
불안 관리4주5회Exposure gradient
기분 안정8주4회Behavioral activation

각 회기 구성

  1. 시작 (5분): HRV, 기분 체크
  2. 카드 실행 (15-60분): 시스템 추천 또는 임상가 override
  3. 종료 (5분): HRV 재측정, OMR

4. Outcome 데이터 입력

4주 회기 종료 후 /outcome.html에서:

  1. Entry Protocol 5 재측정 (모든 측정 다시)
  2. 임상 5축 점수 평가
  3. PHQ-9 / GAD-7 재측정
  4. 자기 보고 (개선 0-10, 만족도 0-10)
  5. 시스템 자동: Adaptive Re-routing (v7) 실행

5. 안전 catch (필수)

위기 신호가 catch되면 시스템이 자동 alert. 임상가는 다음을 확인:
위기 유형Severity즉시 조치
수동적 자살 생각moderate안전 점검
능동적 자살 생각highColumbia SSRS
자살 계획/수단imminent911 + 혼자 두지 마세요
아동학대 공개imminentMA DCF 800-792-5200

MA Mandated Reporter 의무: BCN + PhD = "allied mental health professional" 카테고리. 48시간 내 서면 보고 필수.

6. 데이터 수집 항목

베타 기간 중 다음을 수집:

  • 각 회기마다: 인지 5축, 임상 5축, 바이오마커, OMR 응답
  • 4주마다: Outcome (cognitive_after, clinical_after, 자기 보고)
  • 모든 위기 이벤트: crisis_log 테이블 자동 저장
  • 임상가 override: 이유 필수 기록 (학습 신호 ×2)

데이터는 Supabase (HIPAA 호환)에 저장.

7. 피드백 내용

베타 진행 중, 다음 패턴을 확인:

  • 시스템 추천이 임상적으로 부적합한 경우 (override 빈도)
  • DDE 결과가 클라이언트 임상과 일치하지 않는 경우
  • UI/UX 개선점 (입력 어려움, 결과 가독성)
  • 매뉴얼 항목 missing
  • 새 카드 추가 요청
