운영 순서Workflow

1

요구분석 (고객, 무료)Needs-analysis (client)

고객이 /peak/needs.html에서 15문항. 결과 = 목표 유형(압박형/몰입형/점화형/기복형) + 능력 프로파일 + QEEG 미리보기 + 권위논문. 이 결과(V2 Desire)가 다음 단계로 이어짐.Client does 15 items. Result = type + ability profile + QEEG preview + papers. This V2 carries forward.

2

진단 보고서 — Fischer 레벨 입력Diagnosis — enter Fischer levels

/peak/admin/report.html 진단 탭 → §4 아래 "CNA v7 발달 엔진" 패널.Report → Diagnosis tab → "CNA v7 engine" panel (under §4).

  • 환자 식별자(이니셜) 입력 — 케이스 누적 키Patient ID (initials) — accumulation key
  • Fischer 5축 레벨(L7~13) 입력 — 지속주의·작업기억·정서조절·시간감각·자기인지. Entry Protocol 5 측정값(CPT·N-back·HRV·시간추정·OMR) 환산Fischer 5 levels from Entry Protocol 5 measures
  • "발달 엔진 실행" → 진단·동반진단·신뢰도·최적개입·훈련주수·★누적케이스Run → Dx, confidence, optimal intervention, weeks, ★episodes
3

QEEG/HRV 입력 (선택)QEEG/HRV (optional)

진단 탭 §3 패널. 입력 시 객관분석·통합(MSDI) 탭 추가.§3 panel. Adds objective + integrated (MSDI) tabs.

★ 필수: 손잡이(오른손/왼손/양손) — 전두 알파 비대칭 해석이 갈림. EC(눈감음)/EO(눈뜸) 따로 입력 — 하나만 OK, 둘 다면 알파 반응성도. Mitsar 19ch·HBImed·Neuroguide 실측값.★ Required: handedness; enter EC/EO separately (one OK; both → alpha reactivity). Mitsar/HBImed/Neuroguide.
4

세션 설계Session design

/peak/admin/designer.html. AI 추천 영역 자동선택 + 임상가 임의 추가. 세션수(5/10/15/20) → 세션별 계획 자동 전개. 빠른패턴(매세션NF, 5세션마다QEEG 등). NF 토글 시 SMR↑/高베타↓ 등 자동 배정.AI areas + manual add. Session count → auto plan. Quick patterns (NF each, QEEG every 5). NF toggle assigns protocols.

근거: NF 안정변화 15~40세션(5~10 주관적 개선). SMR 100+ 연구.

5

세션 진행 · 경과 추적Run · track

/peak/admin/progress.html. 매 세션 실측을 채널별로(NF/QEEG/HRV/설문, on/off) 입력 → 기저 대비 변화(④ Adaptive Re-routing). 입력한 채널만 평가. HRV↑면 노출강화, 高베타 잔존이면 각성조절 비중↑, 주관-객관 불일치면 심리훈련 강화.Enter per-channel actuals → change vs baseline. Only entered channels evaluated.

6

진화 확인Verify evolution

/api/health의 episode_count가 케이스마다 오르면 엔진이 학습 중. 케이스 쌓일수록 진단이 정밀해짐(variational update + OMR).Rising episode_count = learning. Diagnoses sharpen over cases.
